## CodePal v1.1.9

这是一个热修版本，修复从旧本地历史库升级时可能导致应用半启动的问题。

### 重点变化

- **修复分析迁移后的启动失败**：旧版 `history.sqlite` 会先补齐新的 token usage source 字段，再创建依赖这些字段的索引。
- **历史库异常不再拖垮整个 App**：如果历史持久化初始化失败，本次启动会降级禁用历史，但主窗口和 IPC 入口仍会继续可用。
- **启动失败日志更明确**：main 启动链路里的未捕获异常会写出明确的 `[CodePal] startup failed` 日志，不再只留下一个 gateway 监听进程。
- **补齐旧数据回归覆盖**：新增旧 token usage schema 迁移测试，以及历史不可用时 IPC 仍可工作的测试。

### 验证

- 已用真实旧 `history.sqlite` 副本复现 v1.1.8 失败并验证修复
- 已验证真实本地数据库迁移后可以恢复 session，并正常打开 IPC
- 已完成本地 unit、lint、build、E2E 完整验证
