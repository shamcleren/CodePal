## CodePal v1.1.10

这个版本修复本地用量历史导入后统计总量异常偏大的问题。

### 重点变化

- **修复分析重复行**：旧版没有 source key 的 token usage 行，如果已经有对应 backfill 行，会在启动迁移时清理。
- **去重 Codex 重复快照**：Codex 重复吐出的 `token_count` 快照改用稳定 usage 签名，不再因为时间戳不同被当作新请求。
- **修正 Codex 缓存 token 口径**：Codex cached input 会和非缓存 input 分开入库，避免总量和费用估算重复计算缓存 token。
- **安全启动清理**：一次性本地分析清理使用索引和临时表，并写入迁移标记，首次清理后不会反复拖慢启动。

### 验证

- 已用真实历史库副本复现 2026-05-18 和 2026-05-19 统计异常偏大的问题
- 已在本地数据库副本上验证清理效果，不直接修改原始数据库
- 已完成 unit、lint、build 和 diff hygiene 验证
